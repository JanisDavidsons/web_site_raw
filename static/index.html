Update 16`th of march, 2020

I have learned a lot about Git.
Version control of this website has already saved me from a disastrous situation a couple of times. There were situations, where I screwed up code so badly, that I would have to start everything from scratch! I`m the person who tries to realize idea as it comes into my mind. Sometimes that brings very very cool stuff into reality, sometimes it destroys everything... Luckily I always have backups available from git! This is where git version control shines. As long as you commit your changes to git, you are safe to experiment and try your craziest ideas. Not worrying about breaking things. You can always recover if things go bad, git remote I'm using is GitHub. You are welcome to check out the history of this website's code if you like here --->my github There are two repositories available. One is called "web_site_raw", which is easy to explore. Another one is called "website_built", which is compiled by parcel for deployment on the server. "website_built" is shared directly to webserver called "Netlify". When I commit changes to this repository, the webserver gets upgraded automatically. Nice! There are more repositories, which I will talk about another time. They have nothing to do with this page. Those are different projects.
